# S2A Manager 使用说明

本文档说明 S2A Manager 管理台的日常使用流程。安装、环境变量和 systemd 托管请看 [部署指南.md](./部署指南.md)。

## 基本概念

- 连接：一个目标 Sub2API 管理后台。每个连接需要填写站点 URL 和 Admin API Key。
- 采集源：用于采集倍率、模型价格和变更记录的源站。支持 Sub2API 和 New API 类型。
- 目标分组：目标 Sub2API 中实际会被修改倍率的分组。
- 目标账号：目标 Sub2API 中实际参与调度的账号。
- 源绑定：把采集源分组绑定到目标分组或目标账号。
- 倍率规则：根据绑定源的生效倍率计算目标分组或账号倍率。
- worker：后台常驻进程，负责自动采集、自动应用规则、上游检测、余额预警和日志清理。

Web 服务和 worker 是两个进程。只启动 Web 可以手动操作管理台，但自动任务必须启动 worker。

## 首次使用

1. 启动 Web 服务并访问 `http://127.0.0.1:15074`。
2. 按页面提示创建第一个管理员账号。
3. 登录后点击左侧或首页的“添加连接”。
4. 填写连接名称、Sub2API 站点 URL 和 Admin API Key。
5. 保存后点击“测试”，确认 S2A Manager 能访问目标 Sub2API 管理接口。
6. 启动 worker：

```bash
npm run worker
```

生产环境建议用 systemd 或 pm2 同时托管 Web 和 worker。

## 推荐配置顺序

建议按下面顺序配置，便于定位问题：

1. 添加并测试 Sub2API 连接。
2. 在“倍率采集”中添加采集源并手动采集一次。
3. 在“分组倍率”或“账号调度”中绑定采集源分组。
4. 保存倍率规则，先手动“保存并应用”验证结果。
5. 确认无误后启用自动采集和 worker 自动应用。
6. 为关键账号配置“上游检测”和余额预警。
7. 在“服务状态”和“日志管理”中观察自动任务结果。

## 连接管理

在左侧连接列表中可以添加、编辑、测试和删除连接。

连接字段：

- 名称：便于识别的本地显示名称。
- Sub2API 站点 URL：目标站点地址，例如 `https://example.com`。
- Admin API Key：目标 Sub2API 管理 API Key。
- 同步策略：手动同步或自动同步。

注意：

- 编辑连接时，Admin API Key 留空表示保持原值不变。
- 删除连接会删除 S2A Manager 本地保存的该连接配置、绑定、规则和采集数据，不会直接删除目标 Sub2API 站点。
- 自动同步需要 worker 常驻运行。

## 倍率采集

进入“倍率采集”页面后，可以维护采集源、执行采集、查看采集倍率和倍率变更记录。

### 添加采集源

点击“添加采集源”，常用字段如下：

- 名称：本地显示名称。
- 源站地址：采集源 URL。
- 源站类型：`Sub2API` 或 `New API`。
- 认证方式：自动登录或手动 Token。
- 邮箱/用户名、密码：自动登录时使用。
- New-Api-User：New API 源站需要指定用户 ID 时填写。
- 采集间隔：worker 自动采集该源站的最小间隔，单位分钟。
- 充值倍率：用于把源站倍率换算为目标站可写入的生效倍率。
- 启用采集：关闭后 worker 不会自动采集该源站。

手动 Token 适合源站无法正常自动登录，或需要使用已有 Session/Cookie/Access Token 的场景。

### 采集与同步

- “测试”用于验证采集源认证和接口是否可用。
- “采集”会立即采集单个源站。
- “采集全部”会对所有启用的采集源执行采集。
- “采集倍率”列表展示源站分组、平台、原始倍率、写入倍率和生效倍率。
- 选择一条倍率和一个目标分组后，可以手动“同步选中倍率”。

采集完成后，如果相关目标分组或账号已经启用倍率规则，worker 会自动尝试应用规则。

### 充值倍率怎么理解

采集源里显示的倍率不一定能直接写入目标站，因为不同站点可能把充值折扣、额度单位或倍率单位处理得不一样。S2A Manager 会用采集源的“充值倍率”换算出生效倍率，后续分组和账号规则使用的都是生效倍率。

常见用法：

- 源站和目标站倍率体系一致：充值倍率填 `1`。
- 源站充值 1 元到账 2 元额度，采集到的倍率相当于已被放大：充值倍率可填 `2`，让生效倍率按比例回落。
- 不确定时先填 `1`，手动采集后对比“原始倍率 / 写入倍率 / 生效倍率”，确认结果符合预期后再绑定规则。

建议先用单个测试分组执行“同步选中倍率”，确认目标 Sub2API 写入值正确，再启用自动规则。

## 分组倍率

“分组倍率”用于维护目标 Sub2API 分组和分组倍率规则。

### 新增或编辑分组

字段说明：

- 分组名称：目标 Sub2API 分组名称。
- 默认倍率：目标分组当前基础倍率。
- 绑定采集源分组：选择一个或多个采集源分组作为规则输入。
- 使用倍率规则：开启后，该分组才会被手动或自动规则改写倍率。
- 倍率规则：首个源倍率、平均源倍率、最低源倍率、最高源倍率或自定义公式。
- 偏移：在规则计算结果上加减固定值，例如 `0.1` 或 `-0.1`。

保存方式：

- 保存：只保存分组、绑定和规则。
- 保存并应用：保存后立即根据规则写入目标 Sub2API 分组倍率。

注意：

- 没有开启“使用倍率规则”的分组不会被自动改写。
- 没有绑定采集源分组时，倍率规则无法计算。
- 如果目标分组在 Sub2API 里被删除或重建，可在“服务状态”执行“清理无效数据”处理残留绑定。

### 倍率规则怎么选

规则输入是当前目标绑定的采集源分组“生效倍率”。只要某个源没有可用倍率，它就不会参与计算；如果所有源都没有可用倍率，规则会报错。

内置规则：

- 首个源倍率：使用第一个有效源倍率。适合只绑定一个源，或明确希望某个源作为主源。
- 平均源倍率：取所有有效源倍率的平均值。适合多个源价格接近，希望平滑波动。
- 最低源倍率：取所有有效源中的最低值。适合跟随最便宜来源。
- 最高源倍率：取所有有效源中的最高值。适合保守定价，避免低估成本。
- 自定义公式：用公式计算更细的策略。

偏移会在规则计算结果之后再叠加。例如平均值是 `1.2`，偏移填 `0.1`，最终写入 `1.3`；偏移填 `-0.05`，最终写入 `1.15`。

结果会四舍五入到 4 位小数，并且必须大于 `0` 且不超过 `100000`。

### 自定义公式

自定义公式可用于分组倍率规则和账号倍率规则。公式只允许数字、变量、函数和数学运算符，不能写中文、比较表达式、三元表达式或任意 JavaScript 代码。

支持的运算符：

```text
+  -  *  /  %  ( )
```

支持的变量：

- `avg`：所有有效源倍率的平均值。
- `first`：第一个有效源倍率。
- `current`：目标分组或账号当前倍率；没有当前值时按 `0` 处理。
- `count`：有效源倍率数量。
- `r0`、`r1`、`r2`：第 1、2、3 个有效源倍率；缺失时为 `0`。
- `rate(0)`、`rate(1)`：按下标读取有效源倍率，等价于 `r0`、`r1`。

支持的函数：

- `min()`：不传参数时取所有有效源最低值；传参数时取参数最低值。
- `max()`：不传参数时取所有有效源最高值；传参数时取参数最高值。
- `sum()`：不传参数时求所有有效源之和；传参数时求参数之和。
- `abs(x)`：绝对值。
- `floor(x)`：向下取整。
- `ceil(x)`：向上取整。
- `round(x)`：四舍五入到 4 位小数。
- `round(x, n)`：四舍五入到 `n` 位小数。
- `clamp(x, min, max)`：把结果限制在指定范围内。

示例公式：

```text
avg
```

使用平均倍率，和“平均源倍率”模式等价。

```text
avg + 0.1
```

平均倍率基础上加 `0.1`。也可以用内置“平均源倍率”加偏移 `0.1` 实现。

```text
min() + 0.05
```

跟随最低源，但保留 `0.05` 的缓冲。

```text
max(avg, 1)
```

使用平均倍率，但最低不低于 `1`。

```text
clamp(avg, 0.8, 2)
```

平均倍率低于 `0.8` 时按 `0.8`，高于 `2` 时按 `2`。

```text
round(avg * 1.08, 4)
```

平均倍率上浮 8%，并保留 4 位小数。

```text
(r0 * 0.7) + (r1 * 0.3)
```

第一个源占 70%，第二个源占 30%。只有两个源的顺序稳定且你明确知道它们的含义时才建议这样写。

```text
count >= 2
```

这种写法不支持。公式不能写比较表达式。需要最低/最高/限制范围时，用 `min()`、`max()` 或 `clamp()`。

实操建议：

- 只绑定一个源时，用“首个源倍率”即可，不必写公式。
- 多源但只想要稳健价格时，优先用 `avg`、`min()`、`max()` 和 `clamp()`。
- 不建议依赖 `r0`、`r1` 做长期策略，因为多个源的顺序可能受源站名称和分组名称影响；确实需要权重时，先确认绑定列表顺序。
- 自定义公式失败后不会写入远端倍率，错误会记录在“日志管理”中。
- 如果上游检测暂停了某个账号，分组规则计算时可能会临时排除该账号关联的源；所有源都被排除时，分组规则会跳过并报错。

### 常见倍率策略

保守跟随成本：

```text
max()
```

适合多个源价格差异大，希望按最高成本定价。

低价吸引但留安全边际：

```text
min() + 0.1
```

适合跟随最低源，但避免过低。

限制异常波动：

```text
clamp(avg, 0.5, 3)
```

适合源站偶尔采集到异常倍率时，限制自动写入范围。

逐步靠近源站价格：

```text
(current * 0.7) + (avg * 0.3)
```

适合不希望目标倍率一次性大幅跳变的场景。注意这会使用当前目标倍率参与计算。

## 账号调度

“账号调度”用于维护目标 Sub2API 账号、账号倍率规则、余额阈值和账号测试。

### 账号字段

常用字段：

- 账号名称、平台、类型、状态。
- 参与调度：控制账号是否参与 Sub2API 调度。
- 并发数、优先级、负载权重。
- 账号倍率。
- 代理 ID。
- 过期时间、过期后自动暂停。
- API Key、Base URL、Credentials JSON、Extra JSON。
- Sub2API 分组。
- 绑定采集源分组。
- 使用账号倍率规则。

保存方式：

- 保存：保存账号信息、绑定和规则。
- 保存并应用：保存后立即按账号倍率规则写入账号倍率。

### 账号测试

点击账号行的“测试”可以选择模型并测试账号可用性。图片模型会显示图片 Prompt 输入框；OpenAI 账号可选择测试模式。

### 余额和预警

账号列表中可以查看余额并设置单账号预警阈值。余额查询支持 Sub2API 管理接口、usage 风格接口以及部分 New API 场景。

余额 Webhook 配置在账号调度页面中维护。启用后，worker 会按余额预警间隔检查账号余额，低于阈值时发送 Webhook。

Webhook 配置项：

- Webhook URL：必须以 `http://` 或 `https://` 开头。
- 冷却分钟：同一账号在冷却期内不会重复发送；填 `0` 表示不做冷却。
- 消息模板：支持变量替换。

余额 Webhook 模板变量：

```text
{{connectionId}}
{{connectionName}}
{{accountId}}
{{accountName}}
{{remaining}}
{{threshold}}
{{unit}}
{{provider}}
{{planName}}
{{checkedAt}}
{{checkedAtIso}}
```

示例模板：

```text
S2A Manager 账号余额预警
连接：{{connectionName}}
账号：{{accountName}} (#{{accountId}})
余额：{{remaining}} {{unit}}
阈值：{{threshold}} {{unit}}
来源：{{provider}} {{planName}}
检测时间：{{checkedAt}}
```

发送逻辑：

- 只有设置了账号阈值、Webhook 已启用且 URL 有效时才会发送。
- 只有余额状态为 `ok` 且余额低于阈值时才会触发。
- Webhook URL 或启用状态变化后，会清空旧的冷却状态。
- 点击“立即检查”会忽略冷却时间，适合验证配置。
- worker 会自动检查，但 Webhook 配置保存后也会标记下一轮尽快检查。

建议：

- 先手动刷新余额，确认余额来源可靠后再配置阈值。
- 先用“测试发送”确认 Webhook 接收端格式正确，再启用正式预警。
- 企业微信 Webhook 会自动使用企业微信文本消息格式。
- 不可靠的余额来源会在日志中记录为 `unsupported`、`invalid` 或 `error`，不会误判为低余额。

### 账号优先级规则

账号优先级规则会按指定分组内账号的倍率排序并写入调度优先级。倍率越低的账号优先级越靠前；倍率相同的账号会保持同一优先级档位。

启用前建议先确认账号分组和倍率都已经整理正确。

使用步骤：

1. 在“调度优先级规则”中启用规则。
2. 选择需要参与排序的目标 Sub2API 分组。
3. 点击“保存规则”。
4. 点击“立即应用”验证写入结果。
5. 后续 worker 在账号倍率规则自动应用后，也会自动尝试应用优先级规则。

排序规则：

- 只处理所选分组内的账号。
- 账号必须能读取到有效倍率，否则会跳过。
- 倍率越低，写入的优先级数字越小。
- 相同倍率使用相同优先级。
- 如果账号同时属于多个选中分组，只会按账号 ID 去重处理一次。

适合场景：

- 低倍率账号成本更低，希望优先调度。
- 多个分组共享一批账号，希望优先级跟随倍率自动调整。
- 账号倍率经常由采集源自动更新，希望优先级也同步更新。

不适合场景：

- 你需要按账号稳定性、供应商额度或手工权重综合排序。当前规则只按账号倍率排序。
- 你需要每个分组拥有完全独立的账号优先级。Sub2API 账号优先级是账号字段，多个分组共享时会互相影响。

## 上游检测

“上游检测”用于持续检测账号可用性，并在失败后自动暂停账号调度。

### 配置规则

点击账号行的“配置规则”或“编辑规则”，填写：

- 启用检测：控制规则是否生效。
- 检测间隔分钟：该账号多久检测一次。
- 连续错误次数：达到阈值后触发暂停。
- 暂停调度分钟：暂停账号参与调度的时长。
- 测试模型：留空时使用 Sub2API 默认测试模型。
- 测试 Prompt：留空时使用默认测试内容。

操作按钮：

- 启用/停用检测。
- 立即检测。
- 暂停中账号可立即恢复调度。
- 删除检测规则。

### 自动暂停和恢复

当检测连续失败达到阈值时，worker 会暂停该账号调度。暂停期间仍会继续按规则检测；检测成功后会自动恢复调度。

如果账号暂停期间绑定了采集源分组，S2A Manager 会临时排除这些源参与对应目标分组的倍率计算，避免故障账号影响分组倍率。

建议：

- 关键账号设置较短检测间隔，例如 1-5 分钟。
- 如果检测间隔设为 1 分钟，worker 运行间隔也建议设为 1 分钟。
- 检测超时和并发在右上角“应用设置”中配置。

## 公告管理

“公告管理”包含手动公告和倍率变化公告规则。

### 手动公告

可创建、编辑、删除公告，并设置：

- 标题和内容。
- 状态：已发布、草稿、已归档。
- 通知方式：静默或弹窗。
- 开始展示时间和结束展示时间。

支持按标题、内容、状态、通知方式和创建时间筛选，并可批量修改状态、展示方式和展示时间。

### 倍率变化公告规则

公告规则用于在目标分组倍率实际变化时自动发布公告。

规则字段：

- 规则名称。
- 是否启用。
- 发布状态和通知方式。
- 适用分组：留空表示全部分组。
- 标题模板和内容模板。

可用变量包括：

```text
{{connectionName}}
{{groupName}}
{{groupId}}
{{oldRate}}
{{newRate}}
{{sourceLabel}}
{{sourceSiteName}}
{{sourceGroupName}}
{{sourceRate}}
{{ruleMode}}
{{ruleExpression}}
{{changedAt}}
{{action}}
```

只有目标分组倍率实际发生变化时，公告规则才会触发。

## 站点设置

“站点设置”用于维护目标 Sub2API 的常用配置。页面按模块分为：

- 站点。
- 注册安全。
- 默认额度。
- 邮件通知。
- OAuth。
- 网关调度。
- 运营支付。

保存前请注意：

- 密码、Secret 等敏感字段留空表示不更新。
- JSON 字段必须是合法 JSON。
- 修改站点安全、OAuth、支付和网关调度配置前，建议先备份目标 Sub2API 配置。

## 服务状态

“服务状态”用于检查系统运行和执行维护操作。

可查看：

- Web 服务在线状态和运行时间。
- 数据库连接状态。
- worker 心跳、最近运行时间、运行间隔、检测超时和检测并发。
- 采集源数量和在线/离线状态。
- 自动同步连接数量。
- 上游检测规则数量。
- 最近任务日志。

### 清理无效数据

当目标分组、账号或采集源在 Sub2API 或 S2A Manager 中被删除后，可能留下本地绑定或规则残留。点击“清理无效数据”可以处理：

- 不存在目标分组的分组绑定和分组倍率规则。
- 不存在目标账号的账号绑定、账号倍率规则和上游检测规则。
- 不存在采集源或源分组的绑定。
- 已删除分组对应的公告目标。
- 已失效的上游暂停倍率源排除记录。
- 源站分组改名后的本地绑定名称更新。

连接不可用或密钥异常时，清理操作会尽量禁用自动规则，保留绑定，便于修复连接后重新启用。

## 日志管理

“日志管理”用于查看和维护任务日志。

### 日志设置

- 日志开关：关闭后不再写入新的任务日志。
- 保存天数：过期日志文件会被清理。
- 记录级别：信息、警告或错误。
- 清理过期：按当前保留天数删除过期日志文件。

日志默认写入项目目录下的 `logs/`。可以通过 `S2A_LOG_DIR` 修改目录。

### 筛选和搜索

支持按以下条件筛选：

- 连接。
- 级别。
- 状态。
- 动作。
- 目标。
- 开始时间和结束时间。
- 关键词。
- 显示条数。

“清空日志”可以清空全部日志或当前筛选连接的日志。清空当前连接日志时，会记录清空标记，不一定删除整个日志文件。

## 应用设置

右上角“应用设置”用于维护 S2A Manager 自身配置。

### Worker 运行配置

- Worker 运行间隔：最小 1 分钟，最大 1440 分钟。
- 上游检测超时：10-300 秒。
- 上游检测并发数：1-20。

这些配置写入数据库后优先于环境变量。环境变量只作为数据库未配置时的默认值。

### 管理员管理

可以添加和删除 S2A Manager 管理员账号。密码至少 6 位。删除管理员前请确认仍保留至少一个可登录账号。

## 常见操作流程

### 自动同步分组倍率

1. 在“倍率采集”添加并测试采集源。
2. 手动采集一次，确认能看到采集倍率。
3. 在“分组倍率”编辑目标分组。
4. 绑定采集源分组。
5. 开启“使用倍率规则”并选择规则模式。
6. 点击“保存并应用”验证倍率写入结果。
7. 确认 worker 已运行，后续采集完成会自动应用已启用规则。

### 自动同步账号倍率

1. 在“账号调度”编辑目标账号。
2. 绑定采集源分组。
3. 开启“使用账号倍率规则”。
4. 选择规则模式和偏移。
5. 点击“保存并应用”验证账号倍率。
6. 查看“日志管理”确认执行结果。

### 账号故障自动暂停

1. 在“上游检测”找到目标账号。
2. 配置检测间隔、连续错误次数和暂停分钟数。
3. 点击“立即检测”验证规则。
4. 确保 worker 常驻运行。
5. 观察 Uptime、最近检测和任务日志。

### 余额 Webhook 预警

1. 在“账号调度”刷新账号余额。
2. 为需要监控的账号填写余额预警阈值。
3. 配置并启用余额 Webhook。
4. 发送测试 Webhook。
5. 确保 worker 常驻运行。

### 排查自动任务异常

1. 打开“服务状态”，确认 worker 在线。
2. 查看最近任务日志是否有失败。
3. 进入“日志管理”，按连接和状态筛选失败日志。
4. 如果提示目标分组、账号或采集源不存在，执行“服务状态”里的“清理无效数据”。
5. 修复连接、密钥、采集源认证或远端 Sub2API 配置后，再手动重试。

## 注意事项

- S2A Manager 会通过 Admin API 修改目标 Sub2API 数据，请先在测试环境验证规则。
- `ENCRYPTION_KEY` 用于解密已保存的 API Key 和源站凭证，生产环境不要随意更换。
- 删除连接、采集源、分组或账号前，先确认是否还有规则依赖。
- 批量公告、自动倍率和自动暂停都建议先手动执行一次，确认日志无异常后再交给 worker。
- 反向代理不要缓存管理台页面，避免登录态或页面内容异常。
- 中文公告按 UTF-8 JSON 发送；乱码通常需要检查反向代理或目标 Sub2API 的编码处理。
